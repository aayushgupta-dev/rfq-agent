# Quick Task 260629-04m: Gated CI/CD (GitHub Actions → Render then Vercel)

## Goal

On push/merge to `main`: run tests → deploy the Render backend → **only if** the backend
deploy reaches `live` AND `/health` responds, deploy the Vercel frontend. The frontend is
never shipped against a stale/dead backend (GitHub Actions `needs:` enforces this in-engine).

Repo: `aayushgupta-dev/rfq-agent`, default branch `main`. This is CI/CD infra, not a product
feature. Keep CLAUDE.md §10/§11 conventions (uv + pnpm, pytest gate, `/health` hermetic).

**Verified before planning:** web pkg `@aerchain/web` build = `next build` (so
`pnpm --filter @aerchain/web build`); `services/ai` uses uv (`pyproject.toml` + `uv.lock`);
`/health` returns `{"status":"ok"}` hermetically; `render.yaml` has no `healthCheckPath`.

Do **not** push and do **not** add anything beyond the four files below.

## Task 1 — Create `.github/workflows/deploy.yml`

**files:** `.github/workflows/deploy.yml` (new)

**action:** Create the file with EXACTLY this content:

```yaml
name: Deploy

on:
  push:
    branches: [main]
  workflow_dispatch:

concurrency:
  group: deploy-main
  cancel-in-progress: false

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12.7"
      - uses: astral-sh/setup-uv@v5
      - name: Install AI service deps
        working-directory: services/ai
        run: uv sync --frozen
      - name: Run AI service tests
        working-directory: services/ai
        run: uv run pytest
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm
      - name: Install web deps
        run: pnpm install --frozen-lockfile
      - name: Build web (compile check)
        run: pnpm --filter @aerchain/web build
        env:
          NEXT_PUBLIC_AI_BASE_URL: https://rfq-agent-ai.onrender.com

  backend:
    needs: test
    runs-on: ubuntu-latest
    env:
      RENDER_API_KEY: ${{ secrets.RENDER_API_KEY }}
      RENDER_SERVICE_ID: ${{ secrets.RENDER_SERVICE_ID }}
    steps:
      - name: Trigger Render deploy (clear cache) and wait for live
        run: |
          set -euo pipefail
          deploy_id=$(curl -fsS -X POST \
            -H "Authorization: Bearer $RENDER_API_KEY" \
            -H "Content-Type: application/json" \
            -d '{"clearCache":"clear"}' \
            "https://api.render.com/v1/services/$RENDER_SERVICE_ID/deploys" | jq -r '.id')
          echo "Triggered Render deploy: $deploy_id"
          for i in $(seq 1 60); do
            status=$(curl -fsS \
              -H "Authorization: Bearer $RENDER_API_KEY" \
              "https://api.render.com/v1/services/$RENDER_SERVICE_ID/deploys/$deploy_id" | jq -r '.status')
            echo "[$i] status=$status"
            case "$status" in
              live) echo "Backend deploy is live."; exit 0 ;;
              build_failed|update_failed|canceled|pre_deploy_failed|deactivated)
                echo "Render deploy failed with status: $status"; exit 1 ;;
            esac
            sleep 15
          done
          echo "Timed out waiting for Render deploy to go live."; exit 1
      - name: Health check
        run: |
          set -euo pipefail
          for i in $(seq 1 20); do
            if curl -fsS https://rfq-agent-ai.onrender.com/health | grep -q '"status"'; then
              echo "Backend /health OK."; exit 0
            fi
            echo "[$i] /health not ready yet..."; sleep 6
          done
          echo "Backend health check failed."; exit 1

  frontend:
    needs: backend
    runs-on: ubuntu-latest
    env:
      VERCEL_ORG_ID: ${{ secrets.VERCEL_ORG_ID }}
      VERCEL_PROJECT_ID: ${{ secrets.VERCEL_PROJECT_ID }}
    steps:
      - uses: actions/checkout@v4
      - name: Install Vercel CLI
        run: npm i -g vercel@latest
      - name: Pull Vercel project settings (production)
        run: vercel pull --yes --environment=production --token=${{ secrets.VERCEL_TOKEN }}
      - name: Build
        run: vercel build --prod --token=${{ secrets.VERCEL_TOKEN }}
      - name: Deploy
        run: vercel deploy --prebuilt --prod --token=${{ secrets.VERCEL_TOKEN }}
```

**verify:** `python -c "import yaml,sys; yaml.safe_load(open('.github/workflows/deploy.yml'))"`
parses without error. Confirm three jobs with `needs:` chain test→backend→frontend.

**done:** File exists, valid YAML, gating chain present.
