<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| /compare/vendors → ExtractionResult[] | Untrusted payload — pydantic validates; vendor count guard blocks oversized arrays |
| compare node → model | Model proposes verdicts; code clamps before emitting — model cannot self-elevate a not_comparable to comparable |
| clarify node → model | Model receives only code-collected flagged fields; cannot invent new items |
| SSE boundary | No pre-clamp verdict ever crosses the SSE boundary (result event emitted only after clamp) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Tampering | Model-proposed comparable verdict over missing field elevated without clamp | mitigate | _apply_verdict_clamp runs before result event; test_clamp_only_downgrades asserts downgrade-only |
| T-04-03-02 | Tampering | Model invents clarification questions for fields not in code-collected list | mitigate | _collect_flagged_fields provides exhaustive list; prompt instructs "exactly N questions for N items"; test_clarification_seeded_by_code asserts count matches |
| T-04-03-03 | Denial of Service | Oversized ExtractionResult[] payload (>5 vendors) | mitigate | ComparisonRequest._check_vendor_count model_validator rejects >5 vendors with 400 |
| T-04-03-04 | Tampering | Raw vendor text passed to comparison (grounding boundary breach, COMPARE-01) | mitigate | _run_align_impl asserts isinstance(e, ExtractionResult) for each input; align node fails loudly on raw text; test_schema_shape confirms no raw_text field on ComparisonResult |
| T-04-03-05 | Tampering | Pre-clamp verdict streamed as final result | mitigate | Clamp and result emit in same try block (CR-01 pattern); test_truncation_error_event + test_refusal_error_event assert no result event on failure paths |
| T-04-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed; all deps verified in Phases 1–3 |
</threat_model>
