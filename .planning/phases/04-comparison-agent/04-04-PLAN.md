    5. NO-NORMALIZATION INSTRUCTION
       "DO NOT convert currencies, split bundles, infer per-item prices from bundled totals, or compute unit prices. Quote verbatim values only. Do NOT set it to a normalized value."

    6. ATTENTION POINTS INSTRUCTION
       "You will receive a list of attention trigger conditions detected by the system. For each trigger, write a buyer-facing summary (summary field on AttentionPoint). Do not add AttentionPoint entries for conditions not in the trigger list."

    7. HUMILITY INSTRUCTION
       "A not_comparable that prevents a misleading comparison is better than a comparable built on incomplete data. When in doubt: use partially over comparable, and not_comparable over partially."

    8. OUTPUT FORMAT
       "Return a single JSON object matching the ComparisonResult schema. Do not include any explanation outside the JSON object."
       Include the ComparisonResult schema fields with types (as a reference block, not as enforced JSON — the structured-output call enforces the schema).

       Example 1: All vendors have present timeline → comparable verdict, reason: "All vendors provided delivery timelines."
