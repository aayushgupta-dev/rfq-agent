<objective>
Close UAT test-8 (severity: major): a deliberate same-field contradiction — two different grand
totals in one vendor response ("all-in program fee USD 1.2M" AND "a total of $950,000, fully
inclusive") — was NOT flagged. The model picked one number, marked total_price `present`, and
silently dropped the other. This violates the product's #1 reliability promise (CLAUDE.md §1/§8:
absence-and-conflict is first-class; the AI never presents one number as definitive when the source
contradicts itself).

Root cause (per .planning/debug/conflicting-not-flagged.md): same-field `conflicting` is 100%
model-judgment — the grounding gate is strictly one-directional (it DOWNGRADES, and by §8 must NOT
invent the dropped claim to upgrade present→conflicting). The model missed it under two prompt
pressures: (1) the total_price bullet frames a single decimal ("extract it as a decimal number"),
and (2) the only conflicting few-shot is a TIMELINE narrative — no numeric/price anchor. Systemic:
no committed sample exercises a same-field total contradiction.

Purpose: Make the extraction prompt reliably surface contradictory grand totals as `conflicting`,
and make that behaviour committed-sample-backed and testable.
Output: Prompt edits (in-place v1 — this is a bugfix within the same contract, see Task 1), one
fixture contradiction, one deterministic test, one live behavioral test, updated prompt doc.

NOT in scope (user explicitly excluded the other UAT gaps): evidence drill-down (test 7), currency
formatting (test 2), regenerate latency (test 3). Do not touch those.
</objective>
