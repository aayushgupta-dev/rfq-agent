State:
  - const { loadedVendors, extractions, setExtraction } = useBuyerContext()

  - const [error, setError] = useState<string | null>(null)

On mount: load rfq from /data/rfq.json (fetch("/data/rfq.json") or import). Set selectedVendor
to first loadedVendor if available.

Session cache check + SSE trigger:
  useEffect deps: [selectedVendor, loadedVendors, rfq]
  if extractions[selectedVendor] → do nothing (cached)
  else if loadedVendors.find(v => v.vendor_name === selectedVendor) && rfq → start SSE
  SSE: for await (const event of streamExtract(vendor, rfq)) { ... }
  On "status" event: update progress.status, advance progress.value by ~20%
  On "result" event: call setExtraction(vendorName, event.payload as ExtractionResult),
    setStreaming(false)
  On "error" event: setError(event.payload.message), setStreaming(false)
  On "done" event: setStreaming(false)

EMPTY STATE (no vendors): Alert with link to /input:
  "Select or load a vendor on the Input screen to begin extraction."

STREAMING STATE: <StreamProgress status={progress.status} value={progress.value} />

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSE stream → React state | ExtractionResult/ComparisonResult cross from AI service to client state |
| sessionStorage → React state | Cached results are deserialized from sessionStorage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06-A | Spoofing | LLM fabricated facts rendered as fact | accept | Grounding gate enforced server-side in Phase 3. The UI renders the grounded ExtractionResult as-is; no client-side fact decision. `unsupported` fields are rendered with that badge — never displayed as `present`. |
| T-05-06-B | Tampering | Comparability verdict manipulation | accept | Comparability clamp is code-enforced server-side (Phase 4). UI renders clamped ComparisonResult from the SSE `result` event — no client re-computation. |
| T-05-06-C | DoS | SSE stream that never closes | mitigate | AbortController tied to component unmount; if user navigates away, reader is released |
</threat_model>
