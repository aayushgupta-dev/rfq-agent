docs/prompts/prompt-04-failure-example.md — PROMPT-04 deliverable (§21 differentiator):
  Per RESEARCH.md Open Question 3 recommendation: use the extraction prompt's humility-bias
  failure as the example (most instructive for the rubric).

  Structure:
  # PROMPT-04 — Prompt Failure Example & Fix

  ## The Failure
  Describe the specific failure: extraction.v1 v0 (before the grounding gate integration)
  would mark fields as `present` with a high-confidence evidence snippet when the snippet
  was actually paraphrased rather than verbatim from the source. Example: model output
  `"status": "present", "evidence": "Vendor offers competitive pricing in the $80K range"`
  when the source text said "We remain flexible on price." — a semantic inference, not a quote.

  ## Why It Happened
  The early prompt did not instruct the model to quote verbatim from source. The model
  "helpfully" generated an evidence-sounding paraphrase that passed a naive substring check
  but failed the rapdf fuzzy gate at threshold 90.

  ## The Fix
  Two-part fix: (1) The extraction prompt was updated to instruct "evidence MUST be a verbatim
  substring of the raw_text — do not paraphrase or summarize", with a few-shot example of a
  verbatim vs. paraphrased quote. (2) The grounding gate in code (gate.py) downgrades
  fabricated or paraphrased snippets to `unsupported` regardless of the model's confidence.
  The fix is code-enforced, not just prompt-enforced (defense in depth).

  ## Versioning Notes
  Describe how the prompt version reflects the fix (version bump from implicit v0 to explicit
  v1 with the verbatim instruction). Note that both the prompt fix AND the code gate are needed —
  neither alone is sufficient. This is the "code disproves model" story.

  ## Evaluation
  Describe how the fix is validated: test_grounding_gate.py runs a fabricated evidence scenario
  through the gate and asserts it gets downgraded to `unsupported` — a permanent regression test.
