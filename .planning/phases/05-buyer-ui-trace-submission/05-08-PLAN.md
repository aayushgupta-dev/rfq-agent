<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vercel → Render | Public internet; HTTPS only; CORS gates the origin |
| Render env → OpenAI API | API key is server-only; never reaches the browser |
| Browser → Render | NEXT_PUBLIC_AI_BASE_URL is the only server knowledge the browser has |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-08-A | Info Disclosure | OPENAI_API_KEY on Render | mitigate | Set via Render environment tab only; never in code or git; .env is gitignored |
| T-05-08-B | Spoofing | CORS misconfiguration allowing unintended origins | mitigate | allow_origins contains only exact strings (localhost + exact Vercel URL); allow_origin_regex covers *.vercel.app previews; dead glob string "https://*.vercel.app" must not appear in allow_origins |
| T-05-08-C | DoS | Render cold start at demo time | mitigate | D-18: warm instance with GET /data/rfq before recording; document in demo-script.md |
| T-05-08-D | Spoofing | LLM fabricated facts in deployed result | accept | Grounding gate enforced server-side (Phase 3); UI renders grounded output as-is; not changeable at deploy time |
</threat_model>
