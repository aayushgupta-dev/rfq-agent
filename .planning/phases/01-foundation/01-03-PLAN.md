## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process env / `.env` -> AI service | OPENAI_API_KEY (a secret) enters the service; the access ping crosses out to the OpenAI API |
| HTTP client -> GET /stream/demo | unauthenticated local request triggers a streamed graph run (Phase 1 is local-only; no auth by design) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | OPENAI_API_KEY in `.env`, ping errors, and startup logs | mitigate | `.env` is gitignored (verified) and never committed (`.env.example` only); the factory, verify_access script, and startup check log/raise only model IDs + pass/fail - code paths never interpolate the API key into any message or log. |
| T-03-02 | Denial of Service | unauthenticated GET /stream/demo runs a graph + (in P3) model calls | accept | Phase 1 demo is local-only and emits hardcoded taxonomy events with no model call on the stream path; no public exposure until P5 (which adds CORS). Rate-limiting/auth is out of scope for a single-buyer local prototype (REQUIREMENTS Out of Scope: auth). |
| T-03-03 | Spoofing | wrong/over-tier model silently used (e.g. gpt-5.5) | mitigate | The tier factory is the single call site reading MODEL_REASONING/MODEL_CHEAP from env; callers pass a tier literal, not a model string - enforcing the gpt-5.4/mini-never-5.5 discipline in one place (D-15). |
| T-03-SC | Tampering | langchain/langgraph/sse-starlette installs | mitigate | Installed in Plan 01 from the version-pinned, RESEARCH-"Approved" stack; reproducibility via committed lockfile. No new installs in this plan. |

The headline risk for this plan is secret leakage of the API key, mitigated by gitignored `.env` + a strict "never log the key" rule on every ping/startup code path.
