Truncation detection (RESEARCH.md Pattern 2 — D-08):
Direct try/except around chain.invoke — LengthFinishReasonError propagates UNCAUGHT through
include_raw=True wrapper (langchain-ai/langchain#29700). Correct pattern:
```python
from openai import LengthFinishReasonError
try:
    raw = chain.invoke(messages)
except LengthFinishReasonError:
    w({"type": "error", "payload": ErrorPayload(
        code="extraction_truncated",
        message="Model output was truncated (finish_reason=length).",
        recoverable=True,
    ).model_dump()})
    return {"error": "truncated"}
```

Refusal detection (RESEARCH.md Pattern 2 — D-08 — addressed in Plan 03-01 + 03-03):
Refusals surface in additional_kwargs["refusal"] on the raw AIMessage, NOT as an exception,
and NOT reliably detectable via str(ValidationError). Use include_raw=True for the refusal
check path only. This is the ONLY correct detection approach — do NOT key refusal off
str(ValidationError) (conflates pydantic schema errors with refusals, violates CLAUDE.md §2).
```python
# ponytail: two-path detection because truncation raises but refusal silently populates a field.
# include_raw=True used solely for refusal inspection; truncation is caught before this runs.
llm_with_raw = get_llm("reasoning").with_structured_output(
    ExtractionResult, method="json_schema", include_raw=True
)
raw_output = llm_with_raw.invoke(messages)  # raises LengthFinishReasonError on truncation
raw_msg = raw_output["raw"]
if raw_msg.additional_kwargs.get("refusal"):
    w({"type": "error", "payload": ErrorPayload(
        code="extraction_refused",
        message="Model refused to extract from this vendor response.",
        recoverable=False,
    ).model_dump()})
    return {"error": "refusal"}
result = raw_output["parsed"]  # validated ExtractionResult
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /extract/vendor | User-supplied VendorResponse.raw_text crosses here; must be bounded and treated as data-only |
| extraction agent → OpenAI API | Vendor raw_text injected as data context in the human message, never the system prompt |
| OpenAI response → SSE client | ground_model gate runs here; only grounded data emitted as result events |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering / Spoofing | Prompt injection via vendor raw_text | mitigate | raw_text injected as human-turn data context only, never concatenated into system prompt; grounding gate rejects snippets not found in source (code-enforced per gate.py) |
| T-03-03-02 | Information Disclosure | Model fabricating grounded claims | mitigate | ground_model() code-enforced in extraction node before any result event emitted (D-07); no LLM-supplied verified flag trusted (CLAUDE.md §2/§8) |
| T-03-03-03 | Denial of Service | Oversized vendor raw_text | mitigate | ExtractionRequest model_validator checks len(vendor.raw_text) <= 200_000 (following VendorGenRequest pattern, RESEARCH.md V5) |
| T-03-03-04 | Information Disclosure | Sensitive data in error SSE events | mitigate | ErrorPayload carries only code + message + recoverable; API key never in any log or error message (factory.py handles key isolation) |
| T-03-03-05 | Tampering | Truncated model output parsed as valid | mitigate | LengthFinishReasonError caught before any parse attempt; error event emitted, node returns early (D-08) |
| T-03-03-06 | Tampering | Refusal mistaken for pydantic ValidationError | mitigate | Refusal detected via additional_kwargs.get("refusal") on raw AIMessage (include_raw=True path) — NOT via str(ValidationError); code-enforced, never model-trusting (CLAUDE.md §2) |
| T-03-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages; all deps from prior phases (RESEARCH.md Package Legitimacy Audit N/A) |
