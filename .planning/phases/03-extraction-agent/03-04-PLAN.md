    2. THE FOUR FLAG STATES (with explicit decision rules + when each applies):
       - present: You found a clear, unambiguous statement and can quote ≥20 characters of verbatim context. Use this state when you are confident.
       - missing: The field has no relevant content in the vendor response. Set value to null, evidence empty. NEVER invent a value.

It does NOT name, list, forbid, or reference any fifth state — the prompt body contains no other status vocabulary whatsoever. The division of labor is: the model outputs one of the four states; a separate code-level gate makes its own decisions after the model returns. Keep these concerns cleanly separated in the prompt.

    3. EVIDENCE INSTRUCTIONS (D-10):
       - For each fact you extract with status=present, unclear, or conflicting: quote a verbatim passage from the vendor response.
       - The snippet must be ≥20 characters and contain ≥3 words of context.
       - Do NOT paraphrase. The quoted text must appear word-for-word in the response.
       - Set source_id to the value provided in the context header (the vendor's source ID).
       - Set char_start=0 and char_end=1 as placeholder values — the system verifies and recomputes exact positions.

    4. RFQ LINE ITEM EXTRACTION (D-01/D-02):
       - The RFQ line items are listed below. For each line item, extract pricing (what the vendor quoted for this specific item) and scope_coverage (what the vendor said they would deliver).

       - Never split a bundled price into per-item amounts — that would be fabrication.

    5. DOCUMENT-LEVEL FIELDS: scope_summary (brief narrative of the vendor's overall offering), commercial_terms (payment, milestones, discounts), timeline (delivery schedule), pricing_structure (if vendor stated a grand total or bundle), total_price (if a specific grand total is separable).

    7. HUMILITY INSTRUCTION (D-09): When in doubt, prefer missing or unclear over present. A confident present on weak evidence is worse than an honest unclear. The buyer would rather see a flag than a fabricated number.

    8. OUTPUT FORMAT: The output must conform exactly to the ExtractionResult JSON schema. All Field values must follow the flag state rules. Do not add fields not in the schema.
